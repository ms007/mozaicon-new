# Elevation

Elevation is never "dramatic." The canvas and preview frames carry a combined
1px border + a two-layer soft shadow so they feel like physical objects resting
on the base surface — but never like they're floating.

## Shadow Scale

Tailwind's shadow utilities are available globally. The resolved values below

## DS Elevation Recipe

The design system prescribes a single, specific elevation treatment for canvas
and preview frames: a hairline border combined with two soft shadow layers.

<div style={{ display: 'flex', justifyContent: 'center', padding: '2rem 0' }}>
  <div
    className="bg-card rounded-md"
    style={{
      width: 180,
      height: 100,
      boxShadow: '0 0 0 1px var(--border), 0 2px 8px rgba(0,0,0,0.15), 0 8px 24px rgba(0,0,0,0.1)',
    }}
  />
</div>

```css
box-shadow:
  /* hairline */
  0 0 0 1px var(--border),
  /* near    */ 0 2px 8px rgba(0, 0, 0, 0.15),
  /* far     */ 0 8px 24px rgba(0, 0, 0, 0.1);
```

## Usage Guidelines

- Prefer `shadow-xs` for controls (buttons, inputs, toggles).
- The three-layer recipe above is reserved for the **canvas frame** and
  **preview panels** — do not apply it to regular UI elements.
- Never use `shadow-lg` or heavier in the main chrome — the app should feel
  flat and tool-like, not layered.

The system

A quiet, functional editor — designed to step out of the way of the pixels.

Logo Designer is a pixel-grid logo editor. The interface it lives in takes after the workbench aesthetic of pro image tools: dark surfaces, a single indigo accent, dense functional type, and controls that are quick to reach but slow to announce themselves. This document codifies that language so every future screen and component feels like it belongs on the same canvas.

Principles

01

The artwork is the hero

Chrome stays neutral so the pixel grid and preview are the loudest things on screen. Accent color appears only on the element a user is currently deciding about.

02

Small, dense, legible

13 px body, 10 px labels, tabular numerics, 4 px spacing grid. Screens fit more context without feeling crowded; every pixel earns its place.

03

One accent, many states

A single indigo accent communicates active tool, active layer, focus ring, slider thumb, and the highlighted segment of any segmented control.

04

Instant, reversible

Every action completes immediately and is stroke-granular in history. UI copy and motion should reinforce that nothing is expensive or irreversible.

05

Keyboard-first parity

Every tool has a single-letter shortcut. Design controls so mouse and keyboard feel equivalent — no UI element should be the only way to do something.

06

Light & dark are peers

Both themes carry the same information density and accent logic. Neither is the "real" one — and neither is more than a token swap away.

Usage rules

Accent marks the one thing the user is deciding about: active tool, active layer, focus ring, selected segment, slider thumb, active swatch outline.

Success is reserved for the layer-visibility dot — "this is on." It never appears as a generic "confirm" color.

Danger is reserved for destructive copy (delete layer, clear canvas). Never used as an accent for color-picker swatches.

Typography

The brand face is Inter Tight — a variable sans used at every weight from 400 to 700. Its tighter tracking and slightly narrower proportions (vs. standard Inter) let 10–13 px labels stay legible without feeling wide. System fallbacks hold the line if the font fails to load. Numerics are tabular everywhere they appear in UI chrome so digits don't reflow as values change.

Elevation

Elevation is never "dramatic." The canvas and preview frames carry a combined 1px border + a two-layer soft shadow so they feel like physical objects resting on the base surface — but never like they're floating. 

box-shadow:
  /* hairline */ 0 0 0 1px var(--border),
  /* near   */ 0 2px  8px rgba(0,0,0,0.15),
  /* far    */ 0 8px 24px rgba(0,0,0,0.10);

Motion

Motion is short (80–200ms) and always on the same ease curve. Transforms only scale small (0.97 button press) or slide segmented pills. Long transitions do not exist — the app is a tool, not a narrative.

Components

The nine parts everything in the editor is built from.

Buttons

Buttons are 28px tall by default (.btn), 24px small (.btn-sm), 20px extra-small (.btn-xs). Icon-only variant is a square. The accent variant is reserved for the primary export action — there is never more than one accent button on screen.

Inputs

Inputs are monospace, tabular-numeric, and match button heights. The small variant (.input-sm, 48px wide) is used for the grid-size field; xs (.input-xs) is used for rotation degrees next to a layer.

Color swatches

A 20×20 square with a 3px radius, a 1px outline inside, and a 2px transparent border that flips to accent when active. The custom-color picker sits last in the row, styled identically.

Segmented control

The pill slides between segments on a 200ms ease-out — the only "noticeable" motion in the system. Used for mutually-exclusive choices of 2–4 options: brush size, mirror mode, smoothing mode, export mode.

Slider

A 4px track with a 14px accent thumb. Always paired with a 30px tabular-numeric value to its right. Debounce the consumer, not the control — the thumb is always responsive even when the output is throttled.

Smoothing 0.35

Layer item

A single row with a visibility dot (success = on, muted = off), a name, and (in the inspector) a row of rotation presets. The whole row is clickable to activate; the dot stops propagation and toggles visibility only.

Tool palette

Six tools, fixed order, single-letter shortcuts. The palette is always visible — never a dropdown, never a hover-to-reveal. Active tool uses accent-muted fill, hover uses the generic bg-hover.

Status bar

A 24px strip at the bottom of the app. All text is 10px monospace, tabular-numeric, --text-muted. Left cluster: zoom, grid size. Right cluster: cursor coordinates, active layer. Never surface anything else here — this strip is for passive readouts only.

Iconography

14 × 14 · 1.3–1.5 stroke · stroke-only, rounded joins.

Every icon in the product is drawn inside a 14×14 viewBox, stroked at 1.3 (tool icons, denser) or 1.5 (header controls, sparser). Fills are reserved for the brand mark and the Fill tool's droplet. Joins and caps are always round.

Rules

Do

  • Use currentColor for stroke and fill
  • 14×14 viewBox · rendered at 14px in controls
  • stroke-linecap="round" + stroke-linejoin="round"
  • Optical-size single glyphs (one concept per icon)
  • Label + shortcut hint in title / aria-label

Don't

  • Mix fill + stroke on the same glyph (brand mark excepted)
  • Use color inside an icon — icons inherit from parent
  • Raster or emoji — everything is vector, everything is system-drawn
  • Add drop shadows, gradients, or bevels
  • Use stroke weights outside 1.3 / 1.5
Voice

Speak like the tool: plainly, without ceremony.

The editor's copy is written like a good keyboard shortcut — short, unambiguous, and forgettable once it's done its job. When in doubt, cut the sentence in half.

Voice qualities

·

Terse, not curt

Two words instead of five — but never at the cost of clarity. "Draw to see preview." not "There is nothing to show yet."

·

Nouns & verbs

Prefer the concrete. "Brush", "Mirror", "Fill", "Export". Avoid adjectives and hedges: no "quickly", no "easily", no "simply".

·

Lowercase values,
Title Case labels

Labels use Title Case ("Tool Options"); values inside them stay lowercase ("filled", "outline", "none", "horizontal"). Dials vs. nameplates.

Lexicon

Use these exact words. They align with what's in the code.

grid
the drawable surface. Never "canvas" in UI copy (canvas is the underlying <canvas> element).
cell
one filled/empty unit of the grid. Not "pixel."
stroke
one complete user interaction (drag, click, resize). The unit of undo.
layer
an independently-smoothed, independently-rotated sheet of cells. Max 3.
smoothing
the 0–1 slider that controls Bézier tension.
mode
"Smooth" vs. "Pixel" for the smoothing engine; "Light / Dark / Auto" for export.
export
the action of producing a file. Never "download", "save", "generate".

Numbers & units

Numbers use tabular-numeric monospace everywhere they appear in UI chrome (status bar, slider values, inputs). Grid sizes are written 16×16 with the math-style multiplication sign (U+00D7), not "16x16" or "16 x 16". Percentages are integers (100%, not 100.0%). Coordinates are row, col — row always first.

Accessibility copy

Every icon-only button carries both a title (for the tooltip) and an aria-label that reads the same thing minus the shortcut. Example: title="Undo (Ctrl+Z)", aria-label="Undo". The visibility dot is a role="switch" with aria-checked.
